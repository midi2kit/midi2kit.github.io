2026年のMIDI 2.0エコシステム — Windows対応・新ハードウェア・Swift開発者への影響

Windows 11がMIDI 2.0をネイティブサポート。DAWのUMP対応、ハードウェアのMIDI-CI採用が加速。Swift開発者にとっての意味と、MIDI2Kitの立ち位置を解説します。

MIDI 2.0エコシステムがクリティカルマスに到達

長年、MIDI 2.0の普及は「もうすぐ」と言われ続けてきました。2026年、ようやくその「もうすぐ」が現実になりました。エコシステムに欠けていたもの — 主要デスクトップOSの両方でのプラットフォームレベルのサポート — が揃ったのです。

Windows MIDI Servicesが2026年2月にGA（正式リリース）を迎えました。Microsoftの新しいスタックは、UMPネイティブトランスポート、マルチクライアントアクセス、マイクロ秒精度のタイムスタンプをWindows 11にもたらします。サードパーティのドライバではなく、MIDI 2.0のためにゼロから構築されたOS標準のサブシステムです。Windows開発者は初めて、ミドルウェアなしでUniversal MIDI Packetの送受信が可能になりました。

Apple側では、CoreMIDIがmacOS 11・iOS 14からMIDI 2.0トランスポートをサポートしています。AppleはUMPサポートを出荷した最初の主要プラットフォームベンダーであり、そのスタックは着実に成熟してきました。CoreMIDIは、すべてのAppleプラットフォームにわたって信頼性の高いUMPパケット配信、エンドポイント列挙、仮想デバイスホスティングを提供しています。

WindowsとmacOSの両方がネイティブMIDI 2.0トランスポートを提供する今、クロスプラットフォームの基盤は整いました。もはや問題は、MIDI 2.0が普及するかどうかではなく、残りのエコシステムがどれだけ早く追いつくかです。

ハードウェアが出荷されている

MIDI 2.0対応ハードウェアは、もはやトレードショーのデモではありません。店頭に並び、スタジオで使われています。

  • KORG Keystage — MIDI-CI Property Exchangeを搭載した最初のコントローラー。multi/polyやwavestateなどの対応シンセに接続すると、パラメータマッピングを自動設定します。接続するだけで、ノブやスライダーが自動的にマッピングされます。
  • Roland A-88MKII — MIDI 2.0 Pianoプロファイルを実装。Synthogy Ivory IIIと連携し、ノートごとのエクスプレッションや高解像度ベロシティに対応します。キーボードはMIDI-CI Discoveryを通じて自身の機能を伝え、ソフトウェアがそれに応じて設定を行います。
  • StudioLogic — 複数のコントローラーがMIDI-CI DiscoveryとProfile Negotiationを標準でサポート。SLシリーズは32ビットベロシティとノートごとのピッチベンドをネイティブで送信します。
  • Rhodes — MK8シリーズは、楽器とコンパニオンソフトウェア間の双方向通信にMIDI 2.0を使用し、リアルタイムのパラメータ同期を実現しています。

パターンは明確です。ハードウェアメーカーは、MIDI 2.0を「後で実装する仕様」として扱っていません。プラットフォームサポートが整ったことで、今まさに出荷しているのです。

DAWが追いついてきた

DAWのサポートは、これまでMIDI標準普及のボトルネックでした。2026年、主要プレイヤーが動き始めています：

  • Cubase 14 — MIDI 2.0からVST3への完全な変換を実現。高解像度ベロシティ（32ビット）とノートごとのコントローラーが、シグナルチェーン全体を流れます。SteinbergがDAWベンダーとVST3仕様のメンテナーの両方であることが、ここでユニークな優位性を生んでいます。
  • Logic Pro — Property Exchangeによる自動デバイスコントロールマッピングをサポート。MIDI 2.0コントローラーが接続されると、Logicは利用可能なリソースを問い合わせ、ユーザーの介入なしにコントロール割り当てを設定します。
  • Studio One — MIDI-CI DiscoveryとProfile Negotiationを処理。DAWがMIDI 2.0デバイスを検出し、互換性のあるプロファイルをネゴシエートし、高解像度通信チャネルを自動的に確立します。

すべてのDAWが対応しているわけではありません — Ableton LiveやFL Studioはまだ実装中です — しかし方向性は明白です。MIDI 2.0は「あったら良い機能」ではなく、競争力のある機能になりつつあります。

Swift開発者にとっての意味

Appleプラットフォームで音楽アプリ、オーディオツール、MIDIユーティリティを開発している方にとって、このエコシステムの変化は直接的な影響があります：

  • デバイスの増加 — MIDI 2.0デバイスが増えるということは、アプリにMIDI 2.0サポートを期待するユーザーが増えるということです。32ビットベロシティ、ノートごとのエクスプレッション、Property Exchangeクエリを送信するコントローラーは、すでにミュージシャンの手元にあります。
  • ユーザーの期待が高まっている — KORG KeystageがCubaseで自動設定されるとき、ユーザーは「なぜあなたのアプリではできないのか」と問うでしょう。MIDI-CI DiscoveryとProperty Exchangeは、「高度な機能」から「期待される動作」へと急速に移行しています。
  • CoreMIDIはトランスポートのみ — AppleのCoreMIDIはパイプを処理します：エンドポイント間のUMPパケットの送受信です。しかし、トランスポート上のMIDI 2.0プロトコルレイヤー — Discovery、Property Exchange、Profile Configuration — は提供されません。自分で実装する必要があります。

CoreMIDIはワイヤーを提供します。MIDI 2.0を本当に便利にするMIDI-CIプロトコルスタックは、開発者に委ねられています。

ここにギャップがあります。準拠したMIDI-CI実装をゼロから構築するということは、SysEx8のパース、トランザクションライフサイクルの管理、チャンク化されたプロパティデータの処理、MUIDの追跡、完全なDiscoveryハンドシェイクの実装を意味します — すべてリアルタイムオーディオコンテキストでスレッドセーフティを維持しながら。

MIDI2Kitの立ち位置

MIDI2Kitは、CoreMIDIのトランスポートレイヤーと、ユーザーやハードウェアが期待する高レベルのMIDI 2.0機能との間のギャップを埋めます。完全なMIDI-CIプロトコルスタックを実装するSwiftライブラリです：

  • MIDI 2.0ファースト設計 — MIDI 1.0 APIのラッパーではありません。MIDI2KitはUMPをネイティブに扱い、後方互換性が必要な場合には双方向のMIDI 1.0/2.0変換を提供します。
  • Swift Package Manager — Package.swiftに1行追加するか、Xcodeの「パッケージの追加」ダイアログを使用するだけです。CocoaPods、Carthage、手動のフレームワーク組み込みは不要です。
  • Discovery + Property Exchange — 自動MUID管理を備えた完全なMIDI-CI Discoveryと、完全なProperty Exchangeトランザクションライフサイクル。AsyncSequenceベースのイベントストリームが、デバイスの出現、機能変更、プロパティ応答をリアルタイムで配信します。
  • Responder API — アプリをMIDI 2.0デバイスとしてネットワークに公開します。型付きリソースの登録、プロパティクエリへの応答、接続ポリシーフィルタリングによるアクセス制御が可能です。仮想楽器、コントロールサーフェス、診断ツールなど、あらゆるMIDI 2.0ホストから認識されるデバイスを構築できます。
  • 602以上のテスト — UMPパース、SysExエンコーディング、Discoveryステートマシン、Property Exchangeトランザクション、並行性の安全性をカバーする77のテストスイート。Swift 6の厳格な並行性が初日から適用されています。

はじめよう

Swift Package ManagerでMIDI2Kitをプロジェクトに追加できます：

// Package.swift
dependencies: [
    .package(
        url: "https://github.com/midi2kit/MIDI2Kit-SDK.git",
        from: "1.0.0"
    )
]

または、Xcodeで File → Add Package Dependencies を選択し、リポジトリURLを貼り付けてください。

わずか数行でMIDI 2.0デバイスを検出できます：

import MIDI2Kit

let client = try MIDI2Client(name: "MyApp")
try await client.start()

for await event in await client.makeEventStream() {
    if case .deviceDiscovered(let device) = event {
        print("Found: \(device.displayName)")
    }
}

Property Exchange、Responder API、UMP変換の詳細については、ドキュメントをご覧ください。
